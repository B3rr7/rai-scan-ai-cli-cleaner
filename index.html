Features

Everything you need to clean up AI agents

Scan, estimate, remove, and roll back — with safety built in at every step.

🔍

Deep scanning

Detects binaries, pip/npm/cargo packages, config dirs, caches, shell startup lines, and systemd units from 100+ known AI CLI tools across your entire home directory.

📊

Disk usage estimates

Calculates how much space each agent consumes with recursive directory sizing and nested-path deduplication. Know what you'll free up before taking action.

🗑️

Recoverable trash

Files move to ~/.rai-scan/trash/ — nothing is deleted immediately. Roll back the latest removal session to restore files, packages, daemons, and shell lines.

🔬

Low-confidence detection

Unknown AI-related data is reported separately using name heuristics and model-file markers (.gguf, .safetensors, .onnx). Never auto-selected for removal.

⚡

Permanent deletion

Use --permanent to delete files directly instead of moving to trash. The purge command empties the trash directory when you're ready.

🛡️

Signed rollback journal

Every removal is recorded in an HMAC-SHA256-signed journal. Tampering with the journal is detected and causes rollback to fail closed — your data stays safe.

Workflow

Three commands to go from scan to cleanup

No config files to edit. No daemons to run. Just install, scan, and choose what to remove.

1

Scan your system

List every detected agent with disk size, confidence level, and the components that were matched. Use --verbose to see every path found.

$ rai-scan list --no-cache --verbose
2

Remove what you don't need

Preview every file, package, shell line, and daemon that will be affected. Confirm with YES to move files to recoverable trash.

$ rai-scan remove claude
# Type YES to confirm
3

Roll back if you change your mind

Restore the last removal session — files come back from trash, packages are reinstalled, daemons re-enabled, shell lines uncommented.

$ rai-scan rollback
rai-scan list --no-cache
$ rai-scan list --no-cache

Safety

Built for safe, recoverable cleanup

rai-scan was designed from the ground up to protect your data — not destroy it.

✓
Recoverable trashFiles are moved to ~/.rai-scan/trash/ and can be restored with a single command. Nothing is deleted without a way back.
✓
Two-factor confirmationEvery removal prints a full preview and requires YES. System-scoped operations also require SYSTEM.
✓
Pre-removal path revalidationEvery path is checked for device, inode, mode, owner, size, and mtime immediately before removal. If anything changed, the operation is blocked.
✗
Root execution refusedInstallation, removal, rollback, and uninstall all refuse to run as root. rai-scan runs with your normal user's permissions only.
✗
Symlinks rejectedSymlinked state directories, journal files, and shell startup files are rejected to prevent symlink-based privilege escalation attacks.
✓
Atomic writes with fsyncAll state files are written atomically using tempfile + os.replace + fsync. Partial writes or crashes never leave corrupt state.
